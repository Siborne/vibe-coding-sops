# i18n: English-first Rules with Bilingual Rationale

## Goal

Make the project English-first for AI-facing content, Chinese-reference for human-facing content.
All 6 existing rules require rewriting; all 6 rationale files require enhancement + translation.

## Strategy: Layered Approach

Proceed in 3 layers — each layer addresses all 6 files before moving to the next, ensuring style consistency within each layer.

**Layer 1:** Enhance all 6 Chinese rationale files (add AI-specific risk section)
**Layer 2:** Rewrite all 6 rule files in English (instructional tone, RFC 2119 keywords)
**Layer 3:** Rewrite all 6 rationale files in English (independent writing, not translation)

- `rules/` — English only. Rules are instructions for AI; no Chinese copy needed.
- `rationale/` — Primary English version (at root).
- `rationale/zh/` — Chinese reference version. Existing Chinese files moved here, then enhanced.

## Layer 1: Chinese Rationale Enhancement

**Source:** Current `rationale/*.md` (Chinese), moved to `rationale/zh/*.md`.

**Change per file:** Replace the short "在 vibe coding 场景下更关键" section (currently ~2-3 sentences) with an expanded "AI 辅助编码的特殊风险" section (~150-300 words):

- How AI specifically violates this rule (concrete example, not abstraction)
- Why AI tends to do this (training data bias, lack of project context, style randomness, etc.)
- What happens if this rule doesn't intercept it

**Sections NOT changed:** 触发场景, 核心洞察, 后果 — these are already solid.

**Workload:** ~1200-1500 new words across all 6 files.

## Layer 2: English Rules Rewrite

**Source:** Current `rules/*.md` (Chinese), used as reference only.

**Principles:**

1. **Instructional voice** — use RFC 2119 keywords (MUST, SHOULD, MUST NOT) to differentiate requirements from recommendations
2. **TL;DR first line** — each file opens with `**Rule:** <one-line summary>`
3. **Shorter, not longer** — keep code examples, trim explanatory prose. AI doesn't need persuasion, it needs clear boundaries
4. **Same structure, reorderable** — core sections stay, but internal order may change for clarity

## Layer 3: English Rationale Rewrite

**Source:** Same core arguments as Chinese rationale, but written fresh in English.

**Principles:**

1. **Independent writing** — not a translation. Re-express the same ideas in idiomatic English.
2. **Same 3-section structure** — Trigger Scenario → Core Insight → AI-Specific Risk → Consequences
3. **Incorporate Layer 1 additions** — the AI-specific risk content from enhanced Chinese files flows into the English versions
4. **English-native examples** — replace Chinese-culture-specific metaphors with English-equivalent ones; keep universal concepts as-is (e.g., bus factor)

## Side Effects

- README.md: needs language badge/note once i18n is in place
- CLAUDE.md "Rules to Load" table: rule file paths unchanged, descriptions may need English refresh
- `docs/变更记录/`: existing change logs are in Chinese — keep as-is (historical record)

## Out of Scope

- Machine translation pipeline or automation
- Any language beyond English + Chinese
- Rule content changes beyond language — no new rules, no rule deletion
