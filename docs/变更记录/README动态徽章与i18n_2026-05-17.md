# README 动态徽章与 i18n

## 基本信息
- **时间**: 2026-05-17
- **修改文件**:
  - `stats.json` (新增)
  - `scripts/generate-stats.sh` (新增)
  - `.claude/rules/auto-update-stats.md` (新增)
  - `README.md` (改为英文版)
  - `README.zh.md` (新增，中文版)

## 根因分析

**问题现象**: README 中规则/技能/理论依据等数量为静态徽章，每次新增文件需手动改数字，容易遗忘导致数据不一致。

**根因**: 项目资产（规则、技能、理论依据、变更记录、计划、规格）分散在多个目录，README 是项目的"仪表盘"，但统计数据靠人力维护，不可靠。

## 修改详情

### 新增: stats.json

**BEFORE**: 不存在  
**AFTER**:
```json
{
  "rules": 9,
  "skills": 1,
  "rationale": 18,
  "changelogs": 10,
  "plans": 5,
  "specs": 5
}
```

> 说明：作为单一数据源，由 `scripts/generate-stats.sh` 自动生成，shields.io 动态徽章读取此文件获取实时统计。

### 新增: scripts/generate-stats.sh

**BEFORE**: 不存在  
**AFTER**: shell 脚本，递归扫描 `rules/`、`skills/`、`rationale/`、`docs/变更记录/`、`docs/superpowers/plans/`、`docs/superpowers/specs/` 目录，统计 `.md` 文件数，写入 `stats.json`。

> 说明：每次新增/删除资产文件后运行此脚本，保持 stats.json 与实际情况同步。

### 新增: .claude/rules/auto-update-stats.md

**BEFORE**: 不存在  
**AFTER**: Claude Code 规则，要求每次修改上述目录中的文件后自动运行 `generate-stats.sh`。

> 说明：防止人类遗忘更新 stats.json，将自动化要求写入项目记忆。

### 修改: README.md → 英文版 + 动态徽章

**BEFORE** (徽章部分):
```html
<img src="https://img.shields.io/badge/rules-9-4caf50.svg" alt="Rules: 9">
```

**AFTER**:
```html
<img src="https://img.shields.io/badge/dynamic/json?url=https://raw.githubusercontent.com/Siborne/vibe-coding-sops/main/stats.json&query=%24.rules&label=rules&color=4caf50" alt="Rules">
```

> 说明：四个动态徽章（rules、rationale、skills、changelogs）从 GitHub raw stats.json 获取数据，push 后自动反映最新数量。同时将内容翻译为英文，作为默认 README。

### 新增: README.zh.md

**BEFORE**: 不存在  
**AFTER**: 中文版 README，保留原中文内容 + 动态徽章，顶部添加 `[English](README.md)` 语言切换链接。

> 说明：GitHub 会根据浏览器语言自动选择 README 变体。中文用户看到中文版，英文用户看到英文版。

## 解决方案

**整体思路**: 单一数据源（stats.json）+ 自动生成脚本（generate-stats.sh）+ 动态徽章（shields.io endpoint badge）+ Claude Code 规则（auto-update-stats.md）。

数据流：修改文件 → `bash scripts/generate-stats.sh` → stats.json 更新 → push → shields.io 拉取新数据 → README 徽章自动更新。
