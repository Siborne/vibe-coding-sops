## 根因分析

用户提出：AI 训练数据覆盖所有风格，不声明偏好会导致每次对话产出不同风格的代码。需要一条规则，强制在开发前声明：风格基准（Google Style vs 阿里巴巴规范）、决策偏好（if-else vs 策略模式）、禁止项。

## 修改详情

### 新增: rules/code-style-declaration.md

定义三个必须声明的维度：风格基准、决策偏好、禁止项。给出 CLAUDE.md 中的声明模板。

### 新增: rationale/code-style-declaration.md

解释核心问题——AI 不是团队，每次回复可能来自训练数据的不同风格区域，不声明就会随机。

### 更新: README.md

- 徽章 rules 计数 5→6
- 规则索引新增第 6 条
- 仓库结构补充 code-style-declaration.md 和 docs/ 目录

### 更新: CLAUDE.md

Rules to Load 表格新增 Code Style Declaration 行。

## 解决方案

规则要求所有新项目首次编码前在 CLAUDE.md 中写入风格声明。存量项目在第一次修改时补充。
