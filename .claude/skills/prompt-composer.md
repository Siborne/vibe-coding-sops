# Prompt Composer

把模糊需求转为可执行的"多步对话脚本"，复制到 Cursor/Claude Code 的 Plan 或 Agent 窗口逐步执行。

## 何时使用

- 任务复杂、涉及多个步骤，不想在一个窗口里全做完
- 需要为执行窗口 AI 精确定义角色和行为边界
- 想把"设计 prompt"和"执行 prompt"拆成两个独立会话

## 工作流程

### 1. 理解需求

用户描述任务后，一次只问一个问题，逐步澄清：

- 目标：最终要产出什么？
- 约束：技术栈、已有代码库、特殊限制？
- 边界：明确不做什么？

直到能清晰描述任务范围，进入下一步。

### 2. 定义执行窗口角色

根据任务性质，为执行窗口 AI 定义角色。包含：

- **身份**：你是一个 xxx 领域的资深工程师
- **技术栈**：擅长 xxx、xxx
- **行为约束**：遵循项目 CLAUDE.md、不确定时标记而非猜测

### 3. 拆分步骤

把任务拆成 3-6 个步骤。每个步骤：

- 独立的、可验证的目标
- 依赖关系明确（Step 2 基于 Step 1 的产出）
- 每步一个 prompt

### 4. 输出脚本

按以下格式输出：

```
## 角色定义

[为执行窗口 AI 定义的角色]

## 对话脚本

按照 Step 1 → Step N 的顺序，逐个复制到 Agent 窗口执行。
每步执行完、确认输出正确后，再进入下一步。

### Step N: [步骤名]

[该步骤的完整 prompt]

---

> 生成的内容需要调整？告诉我要改哪一步。
```

### 5. 提示修改

输出脚本后，提醒用户：

> 以上每个 Step 单独复制到 Agent/Plan 窗口执行。Step 之间确认产出正确再继续。哪个 Step 需要改，告诉我。

## 原则

- **一次一个问题**：澄清阶段不连珠炮
- **步骤独立**：每步产出可验证，不依赖"后面再补"
- **角色精确**：不写"你是一个优秀的工程师"，写"你是一个擅长 FastAPI + PostgreSQL 的 Python 后端"
- **YAGNI**：步骤覆盖用户说的需求即可，不擅自加"最好也做 X"
- **不写代码**：这个 skill 只产出 prompt，不写实现代码
