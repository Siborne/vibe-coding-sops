# 代码变更记录规则

每次修改代码后，必须在 `docs/变更记录/` 目录下创建一个变更记录文件。

## 文件命名

`变更内容简述_日期.md`，例如：`PJAX导航浏览量loading修复_2026-05-11.md`

## 文件内容（必须包含）

### 基本信息
- **时间**
- **修改文件**：列出所有涉及的文件路径

### 根因分析
- 问题现象是什么
- 为什么会发生（技术层面的根因）

### 修改详情
- 每个修改点标注文件路径 + 行号范围
- 贴出修改前（BEFORE）和修改后（AFTER）的代码对比
- 说明这段修改解决了什么问题

### 解决方案
- 修复思路概述

## 代码对比格式示例

    ### 文件: layouts/partials/footer/custom.html (L10-L15)

    **BEFORE:**
    ```js
    旧代码
    ```

    **AFTER:**
    ```js
    新代码
    ```

    > 说明：这里改了什么，为什么要这样改

## 适用范围

所有通过 Edit / Write 工具对项目代码的修改。
